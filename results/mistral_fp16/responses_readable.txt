FLOPs shows theoretical compute needs, while MFU shows actual effectiveness of resource use.

KV cache quantization targets the KV cache, which stores keys and values of attention layers. It's important because the KV cache often consumes lots of memory and acts as a bottleneck for input streams containing lengthy tokens. By implementing KV cache quantization, it's possible to increase throughput and accommodate inputs with longer tokens more efficiently.

KV cache quantization refers to the process of reducing the memory footprint of knowledge vectors (KV) cache used in large language models (LLMs) by quantizing its elements. This is important because as the number of input tokens supported by LLMs increases, so does the memory usage of the KV cache, which can slow down or even crash the model's inference.

NO-RAG ANSWER (73 words):

This technique is used to reduce memory usage, improve performance, and optimize resource utilization in distributed systems that use caching. In traditional KV cache implementations, each value is stored as a floating-point number or a string. However, these data types can consume significant amounts of memory, especially when dealing with large datasets.

Pruning is a powerful technique to reduce the size or complexity of a model by removing redundant components such as individual parameters, neurons, attention heads, or entire layers.

Pruning in neural networks refers to the process of removing unnecessary or redundant connections between neurons during training, in order to reduce the size of the model and improve its efficiency. This is typically done by assigning importance scores to each connection and selectively setting some of them to zero, based on these scores. Structured pruning involves dividing the weight matrix into groups and selecting a subset of weights within each group based on their importance scores.
